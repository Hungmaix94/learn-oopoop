# Backend Engineering Foundation & Architecture Syllabus

This course provides a comprehensive foundation in backend engineering, focusing on database architecture, design patterns, and programming paradigms within an E-commerce context.

## 📦 Module 1: Database Architecture Fundamentals
**File:** `1.database-architecture.md`
*   **Concept:** Understanding Databases as Digital Twins.
*   **Core Principles:** ACID Properties (Atomicity, Consistency, Isolation, Durability).
*   **Design Workflow:** The 4 Phases of Database Design (Analysis -> Conceptual -> Logical -> Physical).
*   **SQL Mastery:** CRUD operations, Indexing strategies, and Performance optimization.
*   **Best Practices:** Normalization (1NF, 2NF, 3NF) and Security (SQL Injection prevention).

## 🛒 Module 2: E-commerce Database Design Masterclass
**File:** `2.ecommerce_db_architecture.md`
*   **Strategy:** Polyglot Persistence (Combining RDBMS, NoSQL, Redis, Elasticsearch).
*   **Schema Design:** Users, Products (SKU/Variants model), Orders, and Inventory.
*   **Advanced Scenarios:**
    *   Handling Concurrency (Pessimistic vs. Optimistic Locking).
    *   Shopping Cart storage strategies.
    *   High-performance Product Filtering & Search.
*   **Scaling:** Partitioning, Sharding, and Indexing best practices.

## ⚖️ Module 3: Programming Paradigms (OOP vs FP)
**File:** `3.fp_vs_oop_ecommerce.md`
*   **Comparison:** Object-Oriented Programming vs. Functional Programming.
*   **Real-world Scenarios:**
    *   Product Management (Encapsulation vs. Separation).
    *   Cart Management (Stateful vs. Immutable).
    *   Discount Logic (Strategy Pattern vs. Higher-Order Functions).
    *   Order Processing (Chaining vs. Composition).

## 🏗️ Module 4: Advanced Object-Oriented Design
**File:** `4.oop_deep_dive.md`
*   **The 4 Pillars:** Encapsulation, Inheritance, Abstraction, Polymorphism.
*   **SOLID Principles:** Writing maintainable and scalable code.
*   **Design Patterns in E-commerce:**
    *   Factory Pattern (Payment Gateways).
    *   Singleton Pattern (Database Connections).
    *   Strategy Pattern (Shipping Calculations).
    *   Observer Pattern (Stock Notifications).
    *   Adapter Pattern (Legacy System Integration).
