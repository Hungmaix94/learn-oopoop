# FPvsOOP Project Workspace

## Overview
This workspace is designed for exploring and comparing **Functional Programming (FP)** and **Object-Oriented Programming (OOP)** paradigms, anchored by the development of a practical **E-Learning Platform**.

The project combines theoretical learning materials with a real-world application implementation (`slides-app`) and visual prototypes.

## Project Structure

### 1. Application: E-Learning Platform (`slides-app`)
*   **Status**: Specification & Prototype Phase
*   **Tech Stack**: Next.js (App Router), TypeScript, Supabase, Tailwind CSS, PayloadCMS.
*   **Goal**: Build a scalable course delivery platform with slide-based lessons, progress tracking, and interactive quizzes.
*   **Documentation**: See [Slides App Specification](./slides-app-spec.md).

### 2. Prototypes (`stitch_course_catalog_listing`)
Contains static HTML/Tailwind CSS prototypes for key UI views:
*   Course Catalog Listing
*   Course Syllabus Details
*   Interactive Slide Content
*   Lesson Slides Overview

### 3. Learning Resources
Structured guides and tutorials covering backend engineering fundamentals:

*   **Database Architecture (`base-be`)**:
    *   Database design principles (ACID, Normalization).
    *   SQL optimization and indexing.
    *   Entity-Relationship modeling.
*   **NestJS (`nest.js`)**:
    *   Core concepts (Modules, Controllers, Services).
    *   Dependency Injection and OOP patterns in Node.js.
*   **Python (`python`)**:
    *   Language basics for backend development.
    *   Data structures and control flow.

## Goals
1.  **Compare Paradigms**: Analyze how FP and OOP approaches differ in solving common backend challenges (e.g., e-commerce logic, state management).
2.  **Build Real Software**: Implement a production-grade E-Learning app using modern React/Next.js patterns.
3.  **Master Backend Foundations**: solidify knowledge of database design and API architecture.
