# NestJS Zero to Hero - Phần 1: Các Khái Niệm Cốt Lõi

> NestJS là một framework Node.js tiến hóa, giúp xây dựng các ứng dụng server-side hiệu quả, đáng tin cậy và dễ mở rộng. Nó sử dụng TypeScript mặc định và lấy cảm hứng mạnh mẽ từ Angular.

---

## 1. Tại Sao Chọn NestJS?

Nếu Express là một chiếc "lá" linh hoạt nhưng lỏng lẻo, thì NestJS là một "kiến trúc" vững chắc.

*   **Kiến trúc rõ ràng**: Ép buộc lập trình viên đi theo cấu trúc Module-based.
*   **TypeScript-first**: Tận dụng tối đa sức mạnh của Decorators và Dependency Injection.
*   **Hệ sinh thái mạnh mẽ**: Hỗ trợ sẵn GraphQL, Microservices, WebSockets, Open API (Swagger).

---

## 2. Kiềng Ba Chân: Module - Controller - Service

Đây là 3 thành phần cơ bản nhất của một ứng dụng NestJS.

### 📦 A. Modules
Làm nhiệm vụ "đóng gói" các thành phần liên quan. Mỗi ứng dụng có ít nhất một **Root Module** (`AppModule`).

```typescript
@Module({
  imports: [],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

### 🎮 B. Controllers
Chịu trách nhiệm xử lý các **Incoming Requests** (HTTP) và trả về phản hồi cho client.

```typescript
@Controller('products')
export class ProductsController {
  constructor(private readonly productsService: ProductsService) {}

  @Get()
  findAll(): string {
    return this.productsService.getAllProducts();
  }

  @Post()
  create(@Body() createDto: any) {
    return 'Tạo sản phẩm mới';
  }
}
```

### 🛠️ C. Providers (Services)
Nơi chứa **Logic nghiệp vụ** (Business Logic) và giao tiếp với Database. Services được inject vào Controllers qua Constructor.

```typescript
@Injectable()
export class ProductsService {
  getAllProducts() {
    return ['Laptop', 'iPhone', 'Keyboard'];
  }
}
```

## 3. Dependency Injection (DI)

NestJS quản lý vòng đời của các object cho bạn. Thay vì khởi tạo `new Service()`, bạn chỉ cần khai báo và Nest sẽ tự động "bơm" nó vào nơi cần.

```typescript
// Trong Controller
constructor(private readonly service: MyService) {}
```

---

## 4. DTO (Data Transfer Object) và Validation

DTO định nghĩa cấu trúc dữ liệu gửi lên. Kết hợp với `class-validator`, Nest sẽ tự động kiểm tra dữ liệu trước khi vào logic.

```typescript
import { IsString, IsNumber } from 'class-validator';

export class CreateProductDto {
  @IsString()
  name: string;

  @IsNumber()
  price: number;
}
```

## 5. Quy Trình Tạo Một Feature Mới

Khi dùng NestJS CLI (khuyên dùng), bạn chỉ cần gõ:

1. `nest g mo products` (Tạo Module)
2. `nest g co products` (Tạo Controller)
3. `nest g s products` (Tạo Service)
