# NestJS Zero to Hero - Phần 2: Middleware, Guards & Interceptors

> Trong phần này, chúng ta sẽ tìm hiểu cách kiểm soát luồng dữ liệu và bảo mật ứng dụng NestJS thông qua các thành phần "trung gian".

---

## 1. Middleware
Middleware là hàm được chạy **trước** khi yêu cầu đến được Controller. Thường dùng cho logging, parsing data.

```typescript
@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    print('Request...');
    next();
  }
}
```

---

## 2. Guards (Vệ sĩ)
Guards chịu trách nhiệm **Xác thực (Authentication)** và **Phân quyền (Authorization)**. Nó quyết định request có được phép đi tiếp hay không.

```typescript
@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    return validateRequest(request); // Trả về true/false
  }
}
```

---

## 3. Pipes (Ống dẫn)
Pipes dùng để **Biến đổi (Transformation)** hoặc **Kiểm tra (Validation)** dữ liệu đầu vào.

*   **Validation**: Nếu dữ liệu không khớp DTO, Pipe sẽ ném ra Exception.
*   **Transformation**: Ví dụ chuyển string '123' thành number 123.

```typescript
@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {
  return this.service.findOne(id);
}
```

---

## 4. Interceptors
Interceptors cho phép bạn can thiệp vào luồng request/response để:
*   Chỉnh sửa dữ liệu trả về (Binding extra logic).
*   Tính toán thời gian xử lý request.
*   Xử lý lỗi toàn cục.

```typescript
@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(map(data => ({ data }))); // Bọc response trong object { "data": ... }
  }
}
```

---

## 5. Luồng xử lý của NestJS (Order of Execution)

Request đi theo thứ tự:
1.  **Middleware**
2.  **Guards**
3.  **Interceptors (Trước khi xử lý)**
4.  **Pipes**
5.  **Controller (Logic xử lý)**
6.  **Interceptors (Sau khi xử lý)**
7.  **Exception Filters** (Nếu có lỗi)
