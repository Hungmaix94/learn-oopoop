# NestJS Zero to Hero - Phần 4: Testing (Unit & E2E Testing)

> Một "Hero" thực thụ không bao giờ đưa code lên production mà thiếu test. NestJS được thiết kế để việc testing trở nên cực kỳ dễ dàng.

---

## 1. Tại Sao NestJS Lại Dễ Test?

Nhờ kiến trúc **Dependency Injection (DI)**. Bạn có thể dễ dàng thay thế (Mock) các Service thật bằng các bản giả (Mocks) để kiểm tra logic mà không cần kết nối Database thật.

---

## 2. Unit Testing (Kiểm Thử Đơn Vị)

Tập trung vào kiểm tra một class hoặc một function duy nhất trong sự cô lập. NestJS sử dụng **Jest** làm framework test mặc định.

**Ví dụ: Test ProductsService**

```typescript
describe('ProductsService', () => {
  let service: ProductsService;

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [ProductsService],
    }).compile();

    service = module.get<ProductsService>(ProductsService);
  });

  it('nên trả về danh sách sản phẩm', () => {
    expect(service.getAllProducts()).toEqual(['Laptop', 'iPhone', 'Keyboard']);
  });
});
```

---

## 3. Mocking Dependencies

Khi test Controller, bạn không muốn chạy logic thực của Service. Bạn sẽ dùng `mock`.

```typescript
const mockProductsService = {
  findAll: jest.fn().mockResolvedValue([{ name: 'Test Product' }]),
};

@Module({
  controllers: [ProductsController],
  providers: [
    {
      provide: ProductsService,
      useValue: mockProductsService, // Inject bản mock thay vì bản thật
    },
  ],
})
```

## 4. E2E Testing (End-to-End)

Kiểm tra toàn bộ luồng của ứng dụng từ lúc nhận HTTP Request cho đến khi trả về kết quả. NestJS sử dụng thư viện `supertest`.

```typescript
describe('AppController (e2e)', () => {
  let app: INestApplication;

  beforeEach(async () => {
    const moduleFixture: TestingModule = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = moduleFixture.createNestApplication();
    await app.init();
  });

  it('/products (GET)', () => {
    return request(app.getHttpServer())
      .get('/products')
      .expect(200)
      .expect(['Laptop', 'iPhone', 'Keyboard']);
  });
});
```

---

## 5. Các Lệnh Chạy Test Quan Trọng

*   `npm run test`: Chạy toàn bộ Unit Test.
*   `npm run test:watch`: Chạy test ở chế độ quan sát (tự chạy lại khi sửa code).
*   `npm run test:cov`: Xem báo cáo độ bao phủ code (Coverage report).
*   `npm run test:e2e`: Chạy các bài test luồng đầu cuối.
