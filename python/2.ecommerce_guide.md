## 2. Làm Việc Với Cơ Sở Dữ Liệu (Database Interaction)

Một Backend E-commerce phải giao tiếp tốt với cả SQL và NoSQL như đã thiết kế trong phần Architecture.

### 🐘 A. PostgreSQL với Python (SQLAlchemy & Psycopg)

Sử dụng **SQLAlchemy (ORM)** để tương tác an toàn và Pythonic.

**Ví Dụ: Xử Lý Transaction Đặt Hàng (ACID)**

```python
from sqlalchemy import create_engine, Column, Integer, String, Float, ForeignKey
from sqlalchemy.orm import sessionmaker, declarative_base

Base = declarative_base()
engine = create_engine('postgresql://user:pass@localhost/ecommerce_db')
Session = sessionmaker(bind=engine)

class Order(Base):
    __tablename__ = 'orders'
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer)
    total = Column(Float)

def create_order_transaction(user_id, items, total_amount):
    session = Session()
    try:
        # 1. Bắt đầu Transaction
        with session.begin():
            # 2. Trừ tồn kho (Giả sử có logic check tồn kho)
            # update_inventory(session, items) 
            
            # 3. Tạo đơn hàng
            new_order = Order(user_id=user_id, total=total_amount)
            session.add(new_order)
            
            # 4. Commit tự động nếu không có lỗi
        print("✅ Đặt hàng thành công!")
    except Exception as e:
        # 5. Rollback tự động nếu có lỗi (Nhờ context manager)
        print(f"❌ Lỗi: {e}")
        # session.rollback() # (Đã được xử lý bởi session.begin())
    finally:
        session.close()
```

### 🍃 B. MongoDB với Python (PyMongo & Motor)

Dùng cho Catalog sản phẩm đa dạng thuộc tính.

**Ví Dụ: Tìm Kiếm Sản Phẩm Theo Nhiều Tiêu Chí**

```python
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017/')

    """
    Search linh động: criteria = {"attrs.ram": "16GB", "price": {"$lt": 1000}}
```

### ⚡ C. Redis với Python (Redis-py)

Dùng cho Caching và Giỏ hàng.

**Ví Dụ: Pipeline (Batch Operations) Tăng Tốc Độ**

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

def add_to_cart_batch(user_id, items):
    """
    Thêm nhiều món vào giỏ hàng chỉ với 1 round-trip network.
    """
    pipe = r.pipeline()
    key = f"cart:{user_id}"
    
    for item_id, quantity in items.items():
        pipe.hset(key, item_id, quantity)
        
    pipe.expire(key, 86400) # Hết hạn sau 1 ngày
    pipe.execute() # Thực thi tất cả lệnh cùng lúc
```

## 3. Python OOP vs FP Trong E-commerce

Python là ngôn ngữ đa mô hình (Multi-paradigm). Bạn có thể viết Class hoặc viết Function tùy bài toán.

### 🏛️ OOP Style (Data Classes)
Tốt cho việc định nghĩa Schema dữ liệu rõ ràng.

```python
from dataclasses import dataclass
from typing import List

@dataclass
class CartItem:
    sku: str
    price: float
    quantity: int

    @property
    def total(self):
        return self.price * self.quantity

@dataclass
class Cart:
    items: List[CartItem]

    def get_grand_total(self):
        return sum(item.total for item in self.items)

# Sử dụng
item1 = CartItem("IPHONE15", 1000.0, 2)
my_cart = Cart([item1])
print(my_cart.get_grand_total()) # 2000.0
```

### 🧪 FP Style (List Comprehensions & Map/Filter)
Tốt cho xử lý, biến đổi dữ liệu (Data Transformation).

```python
orders = [
    {"id": 1, "status": "PAID", "amount": 100},
    {"id": 2, "status": "PENDING", "amount": 50},
    {"id": 3, "status": "PAID", "amount": 200},
]

# Tính tổng tiền các đơn đã thanh toán (PAID)
# Phong cách Functional: Ngắn gọn, không side-effect
total_revenue = sum(
    order["amount"] 
    for order in orders 
    if order["status"] == "PAID"
)

# Sử dụng map/filter
paid_orders = list(filter(lambda o: o["status"] == "PAID", orders))
amounts = list(map(lambda o: o["amount"], paid_orders))
```

## 4. Xử Lý Bất Đồng Bộ (AsyncIO) - Vũ Khí Hiệu Năng

Với Python hiện đại (3.7+), **Async/Await** là bắt buộc cho các IO-bound task (Gọi DB, Gọi 3rd party API) để chịu tải cao.

**Ví Dụ: Gửi Email Marketing cho 1000 Users**
Nếu dùng vòng lặp thường, user thứ 1000 phải đợi 999 người trước -> Cực lâu. 
Dùng Asyncio -> Gần như song song.

```python
import asyncio
import time

async def send_email(user_id):
    # Giả lập độ trễ mạng (IO bound)
    await asyncio.sleep(1) 
    print(f"📧 Sent to user {user_id}")

async def main():
    start = time.time()
    
    # Tạo 10 tasks chạy đồng thời
    tasks = [send_email(i) for i in range(10)]
    
    # Chờ tất cả hoàn thành
    await asyncio.gather(*tasks)
    
    end = time.time()
    print(f"⏱️ Tổng thời gian: {end - start}s") 
    # Kết quả: ~1 giây (Thay vì 10 giây nếu chạy tuần tự)

# Chạy event loop
# asyncio.run(main())
```

## 5. Mẹo Tối Ưu Python (Tips & Tricks)

1.  **Sử dụng Set cho việc tìm kiếm:**
    -   `item in list` tốn O(N). `item in set` tốn O(1).
    -   Nếu cần check blacklist user, hãy dùng Set.

2.  **Generators thay vì List:**
    -   Khi xử lý file log 10GB, đừng `readlines()` vào RAM. Hãy dùng Generator (yield) để đọc từng dòng.

3.  **Type Hinting (Pydantic):**
    -   Python là Dynamic Type, nhưng dự án lớn cần Type Safety.
    -   Sử dụng **Pydantic** để validate dữ liệu đầu vào API cực mạnh.

    ```python
    from pydantic import BaseModel, PositiveInt

    class CreateOrderRequest(BaseModel):
        user_id: int
        items: list[str]
        total: PositiveInt # Tự động lỗi nếu số âm
    ```
